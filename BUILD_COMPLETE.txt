## ğŸ” Authentication Details

### User Registration
- Fields: Name, Email, Password
- Validation: Email format, password length (6+ chars)
- Password: Hashed with bcrypt (10 rounds)
- Response: JWT token + user info

### User Login
- Fields: Email, Password, Role (user/admin)
- Validation: Email and password required
- Response: JWT token + user info
- Token expiry: 7 days

### Admin Login
- Same as user login but role = "admin"
- Admin-only routes are protected
- Can create/view health tips

## ğŸ“ˆ API Endpoints (17 Total)

### Authentication (3)
- `POST /api/auth/register` - Register user
- `POST /api/auth/login` - Login (user/admin)
- `GET /api/auth/profile` - Get user profile

### Medicine (3)
- `POST /api/medicine` - Add medicine
- `GET /api/medicine` - Get medicines
- `DELETE /api/medicine/:id` - Delete medicine

### Appointment (3)
- `POST /api/appointment` - Schedule appointment
- `GET /api/appointment` - Get appointments
- `DELETE /api/appointment/:id` - Delete appointment

### Prescription (3)
- `POST /api/prescription/upload` - Upload image
- `GET /api/prescription` - Get prescriptions
- `DELETE /api/prescription/:id` - Delete prescription

### Admin (3)
- `POST /api/admin/health-tips` - Create tip
- `GET /api/admin/health-tips` - Get all tips
- `POST /api/admin/seed-health-tips` - Seed defaults

### System (1)
- `GET /api/health` - Health check
